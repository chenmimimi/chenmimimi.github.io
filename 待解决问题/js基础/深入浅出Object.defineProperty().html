从对象的定义和赋值开始，我们深入Object.defineProperty()这个方法

https://www.jianshu.com/p/8fe1382ba135

这篇文章将按照下面的几点展开：

对象的定义与赋值

经常使用的定义与赋值方法obj.prop = value或者obj[‘prop’] = value

Object.defineProperty()语法说明

Object.defineProperty()的作用就是直接在一个对象上定义一个新属性，或者修改一个已经存在的属性

  1. obj 需要定义属性的当前对象
  2. prop 当前需要定义的属性名
  3. desc 属性描述符

一般通过为对象的属性赋值的情况下，对象的属性可以修改也可以删除，但是通过Object.defineProperty()定义属性，通过描述符的设置可以进行更精准的控制对象属性。

属性的特性以及内部属性

javascript有3种类型的属性

  1. 命名数据属性：拥有一个确定的值的属性。这也是最常见的属性
  2. 命名访问器属性：通过getter和setter进行读取和赋值的属性
  3. 内部属性：由Javascript引擎内部使用的属性，不能通过Javascript代码直接访问到，不过可以通过一些方法间接的读取和设置。比如，每个对象都有一个内部属性[[prototype]],你不能直接访问这个属性，但是可以通过Object.getPrototypeOf()方法间接的读取到它的值，虽然内部属性通常用一个双吕括号包围的名称来表示，但这其实不是它的名字，只是一种抽象操作，是不可见的，根本没有上面两种属性有的那种字符串类型的属性
属性描述符

通过Object.getPrototypeOf()方法定义的属性，有两种形式，且不能混用，分别为数据描述符，存取描述符，下面分别描述下两者的区别：

数据描述符 — 特有的两个属性（value, writable）

描述符里面的属性没有设置，会使用默认值

存取描述符 — 是由一对getter, setter函数功能来描述的属性

get：一个给属性提供getter方法，如果没有getter则为undefined,该方法返回值被用作属性值，默认为undefined
set：一个给属性提供setter方法，如果没有setter则为undefined,该方法接受唯一参数，并将该参数的值重新分配给该属性，默认为undefined

数据描述符和存取描述符都有以下描述符：

  1. configurable描述属性是否配置，以及可否删除
  2. enumerable描述属性是否会出现for in 或者Object.keys()的遍历中

2种等价
不变性
1. 对象常量

结合configurable: false 和writable: false 就可以创建一个真正的常量属性（不可以修改，不可以重新定义和删除）

2. 禁止扩展

如果你想禁止一个对象添加新属性并且保留原有属性，就可以使用Object.preventExtensions();

3. 密封

Object.seal()会创建一个密封对象，这个方法实际上会在一个现有对象上调用Object.perventExtensions()并且把所有现有属性标记为configurable: false

所以密封之后不仅不能添加新属性，也不能重新配置或者删除任何现有属性，但是可以对现有属性重新赋值

4. 冻结

Object.freeze()会创建一个冻结对象，这个方法实际上会在一个现有对象上调用Object.seal()并且把所有现有属性标记为writable: false, 这样，我们也无法将其重新赋值

你可以深度冻结一个对象，具体方法为，首先这个对象上调用Object.freeze()，然后遍历它引用的所有对象，并在这些对象上调用Object.freeze()。但是这么做一定要小心，因为可能会无意中冻结其他共享对象
